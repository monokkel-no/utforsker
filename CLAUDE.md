## Commands

```bash
# Development
pnpm dev          # Start all dev servers in parallel (web app on :3000)
pnpm build        # Build all apps and packages
pnpm format       # Format with Biome

# Checks (run before committing)
pnpm check        # Runs Biome + Knip + per-package tsc in parallel
pnpm check:biome  # Lint/format check only
pnpm check:knip   # Unused exports check (ignores routeTree.gen.ts)

# Web app only
cd apps/web
pnpm dev          # Vite dev server on :3000
pnpm test         # Vitest unit tests
pnpm check        # Biome check scoped to this app
```

Running tests for a single file: `pnpm --filter @repo/web exec vitest run src/path/to/file.test.ts`

## Architecture

This is a **pnpm monorepo** (Node ≥ 25 required) with a single web application and several shared packages. All packages are `type: "module"` ESM and export directly from `src/` with no build step (except `@repo/ui-theme`).

### Package layout

| Package | Purpose |
|---|---|
| `@repo/core` | `UserRole` enum (Teacher, Student) — shared domain types |
| `@repo/chat` | `ChatView` component + `MessageContent` (markdown/LaTeX/code) + `Message` type + LLM API calls |
| `@repo/knowledge-graph` | `KnowledgeGraph` component built on `@xyflow/react` |
| `@repo/course-browser` | `CourseTree` file-tree component + `coursesPath.ts` (server-only) |
| `@repo/file-editor` | Monaco editor wrapper |
| `@repo/ui-components` | Thin MUI v7 wrappers (Button, TextField, UIProvider, …) |
| `@repo/ui-theme` | MUI theme + CSS variables, has a build step (`tsx src/build.ts`) |
| `@repo/utils` | Shared utilities |
| `@repo/typescript-config` | `base.json` (NodeNext) and `react-library.json` (+ react-jsx) |

### Web app (`apps/web`)

**Framework**: TanStack React Start (Vite + SSR) + TanStack Router v1 (file-based routing).

**Routing** — files in `apps/web/src/routes/`:
- `__root.tsx` — root layout with `ThemeProvider` + `UIProvider` + devtools
- `index.tsx` — `/`
- `$userRole/index.tsx` — `/:userRole` — role selection (Teacher/Student via `UserRole` enum)
- `$userRole/courses.$.tsx` — `/:userRole/courses/*` — three-pane: course tree / chat / knowledge graph; splat param is `_splat`
- `-api/` — server functions (the `-` prefix hides this directory from the router)

`routeTree.gen.ts` is auto-generated by the Vite plugin on every save — never edit it manually.

**Environment**: `.env` lives at the **workspace root** (loaded by Vite via `dotenv.config({ path: "../../.env" })`). Required keys: `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `HUGGING_FACE_TOKEN`.

### Import conventions

- Internal packages: `import { Foo } from "@repo/chat/Foo.tsx"` (use `.tsx` for components, `.js` for plain TS modules)
- Workspace deps in `package.json`: `"@repo/chat": "workspace:*"`
- Shared dep versions: `"react": "catalog:"` (centrally versioned in `pnpm-workspace.yaml`)
- In `apps/web`, `@` resolves to `./src`

### Styling

- **MUI v7** wrapped by `@repo/ui-components`; theme applied via `ThemeProvider` + `UIProvider` in root layout
- **Vanilla Extract** for CSS-in-JS (compiled at build time via `@vanilla-extract/vite-plugin`)
- **CSS variables** defined and exported from `@repo/ui-theme`
